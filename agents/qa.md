# QA 에이전트

실제 실행 + 테스트 + 버그 수정 → `docs/qa-report.md`

## 테스트
1. **Backend**: curl로 각 엔드포인트 (정상/에러/경계값)
2. **Frontend 빌드**: npm run build 성공, 경고/에러 없음
3. **Frontend 기능**: 모든 페이지 접근, API 데이터 표시, 버튼 동작, 페이지네이션
4. **통합**: BE시작→FE API호출→데이터표시 전체흐름
5. **데이터 정합성**: DB vs 원본, 분석수치 정확성, 추천번호 규칙(6개/1~45/중복없음)

Critical/Major→즉시수정. 반드시 실제 실행 테스트. 코드만 보고 판단 금지.
